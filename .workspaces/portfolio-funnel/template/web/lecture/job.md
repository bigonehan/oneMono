# graphql_relay
- Node/Edge/Connection/PageInfo와 전역 ID 인코딩 규칙을 일관 적용한다.
- 도메인별 API가 동일한 Relay 페이징 규약을 따르도록 유지한다.
> 도메인 모듈별 Relay 스키마 적용 상태를 대조한다.
> 누락된 공통 규격을 모듈 단위로 보완한다.

# dataloader
- N+1 조회를 방지하도록 도메인별 배치 조회를 적용한다.
- 기존 동작 호환성을 유지하며 조회 경로를 정리한다.
> 조회 패턴을 점검해 반복 쿼리 구간을 식별한다.
> DataLoader 배치 조회로 교체 후 결과 일관성을 재검증한다.

# ssr
- main/intro/detail/user 페이지에서 SSR 초기 데이터 로드를 보장한다.
- API 오류 노출과 hydration 가드를 공통 규칙으로 적용한다.
> 페이지별 데이터 로드·에러 처리 경로를 점검한다.
> 공통 쿼리 조합과 가드 로직을 적용해 렌더 안정성을 보완한다.
