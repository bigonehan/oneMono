# astro
- 앱 엔트리와 페이지 라우팅 기준은 Astro로 고정한다.
- 구조 시각화 화면 진입 경로를 단일 라우트 규약으로 유지한다.
> Astro 기본 엔트리와 라우트 파일을 구성한다.
> 구조 시각화 초기 화면이 라우트로 연결되는지 확인한다.

# zustand
- 상태 관리는 전역 단일 Zustand 스토어로 통합한다.
- 상태 전이 키는 스캔/파싱/그래프/렌더 단계와 동기화한다.
> 스토어 상태/액션 타입을 정의한다.
> Pixi 렌더 계층과 상태 참조 경로를 연결한다.

# cwd
- 루트 탐색 기준은 실행 시점 현재 작업 디렉터리로 고정한다.
- 탐색 대상은 모노레포 루트와 `domains/domain`으로 제한한다.
> cwd에서 루트 판별 로직을 실행한다.
> 루트 판별 결과를 스토어에 기록한다.

# domains_domain
- `domains/domain` 하위만 수집 대상으로 사용한다.
- 탐색 대상 외 경로는 스캔하지 않는다.
> 대상 경로를 해석하고 존재 여부를 검증한다.
> 언어별 수집 파이프라인에 경로 목록을 전달한다.

# js_ts
- export/함수 시그니처만 추출하고 함수 본문은 저장하지 않는다.
- 파싱 실패 파일은 오류 목록에 기록하고 전체 처리는 계속한다.
> JS/TS 파일을 수집한다.
> 모듈 구조 정보를 추출해 중간 데이터로 저장한다.

# rust
- `fn`, `impl`, `mod` 구조를 추출하고 pub 여부를 보존한다.
- 파싱 에러 파일은 건너뛰고 에러 상태를 스토어에 기록한다.
> Rust 소스 파일을 수집한다.
> 심볼 구조를 추출해 카드 호환 데이터로 변환한다.

# pixi
- 기본 시각화 렌더러는 PixiJS를 사용한다.
- 카드에는 모듈명과 함수 리스트를 함께 표시한다.
> 카드 레이아웃 좌표를 계산한다.
> Pixi 캔버스에 카드와 함수 목록을 렌더링한다.

# shadcn_ui
- `@ui/shadcn` 컴포넌트가 있으면 동일 역할에서 우선 사용한다.
- 커스텀 UI는 shadcn 대체가 불가한 경우에만 사용한다.
> 사용 가능한 shadcn 컴포넌트를 확인한다.
> 해당 컴포넌트를 카드 주변 UI에 우선 연결한다.

# stale
- 동일 경로 재스캔 시 이전 결과를 교체한다.
- stale 데이터는 제거해 화면과 저장 상태 불일치를 방지한다.
> 변경 감지 기준으로 재스캔 범위를 계산한다.
> 갱신 데이터 반영 후 이전 스냅샷을 정리한다.
