# info
name : feature_reader_component_mobile_md_expo_1_doc_list_2_doc_3_doc_reader_4_option_tab_zustand_flashlist_ui_react_native_elements_maestro
description : feature/reader component를 이용해서 Mobile에서 md파일을 리더기처럼 읽는 Expo 앱을 만든다. 기본 화면은 1) doc list 2) 선택한 doc 소개 3) doc reader 4) option 화면이고 하단 tab으로 전환한다. 상태관리는 zustand, 목록은 FlashList, UI는 React Native Elements, 테스트는 Maestro를 사용한다.
spec : expo, react native, react navigation, zustand, @shopify/flash-list, react native elements, maestro
path : /home/tree/home/template/mobile/reader

# features
- node_package_workspace

# rules
- 앱은 Expo 기반 React Native 프로젝트로 구성한다.
- 기본 화면 구성은 `doc list`, `doc 소개`, `doc reader`, `option` 4개 탭으로 고정한다.
- 화면 전환은 React Navigation의 하단 tab 네비게이션을 기준으로 설계한다.
- 전역 상태의 단일 소스는 zustand store로 유지한다.
- 문서 목록 렌더링은 반드시 `@shopify/flash-list`를 사용한다.
- 공통 UI 컴포넌트와 화면 기본 요소는 React Native Elements를 우선 사용한다.
- 문서 데이터는 markdown 문서를 읽기 가능한 구조로 정규화해서 사용한다.
- 선택된 문서 상태는 목록, 소개, 리더 화면에서 일관되게 공유한다.
- 리더 화면은 문서 읽기 흐름에 집중하도록 본문 표시를 우선한다.
- 옵션 화면은 리더 동작과 표시 설정을 변경하는 진입점으로 사용한다.
- 테스트 기준 사용자 흐름은 Maestro 시나리오로 정의한다.
- 워크스페이스 구조에서는 앱 패키지와 공통 설정의 경계를 명확히 유지한다.

# constraints
- 프로젝트는 mobile 전용 UX를 기준으로 구현하며 desktop 전용 레이아웃을 요구하지 않는다.
- 상태관리는 zustand 외 별도 전역 상태관리 라이브러리를 추가하지 않는다.
- 문서 목록 컴포넌트는 FlatList가 아니라 FlashList를 사용해야 한다.
- 기본 UI 계층은 React Native Elements 기준으로 유지하며 임의의 대체 UI 프레임워크를 도입하지 않는다.
- 네비게이션 구조는 4개 하단 탭을 유지해야 하며 다른 기본 엔트리 구조로 대체하지 않는다.
- 문서 읽기 대상은 markdown 형식이어야 한다.
- 문서 선택 전에는 소개/리더 화면이 비정상 상태로 깨지지 않도록 안전한 빈 상태를 제공해야 한다.
- 옵션 변경은 앱 재시작 없이 현재 세션 상태에 반영되어야 한다.
- Expo 환경에서 동작 가능한 라이브러리와 구현 방식만 허용한다.
- 테스트는 Maestro 실행이 가능한 사용자 흐름 단위로 준비해야 한다.

# domains
## app_shell
### states
- 앱 초기화 여부
- 하단 탭 현재 선택 상태
- 문서 데이터 로드 완료 여부
- 전역 오류 노출 여부
### action
- 앱 시작 시 초기 데이터와 기본 설정을 로드한다.
- 하단 탭을 전환한다.
- 초기 진입 화면을 렌더링한다.
- 전역 오류 또는 빈 상태 화면을 표시한다.
### rules
- 앱의 기본 내비게이션 엔트리는 4개 하단 탭으로 구성한다.
- 초기화가 끝나기 전에는 불완전한 화면 대신 명시적인 로딩 상태를 제공한다.
- 전역 상태 의존 화면은 app shell 초기화 결과를 기준으로 렌더링한다.
### constraints
- 스택 중심 구조로 대체하지 않고 tab 중심 구조를 유지해야 한다.
- 초기화 실패 시 앱이 종료되지 않도록 복구 가능한 상태를 제공해야 한다.

## doc_list
### states
- 전체 문서 목록
- 문서 목록 로딩 상태
- 현재 선택된 문서 id
- 목록 비어 있음 상태
### action
- markdown 문서 메타데이터 목록을 불러온다.
- FlashList로 문서 목록을 렌더링한다.
- 문서를 선택한다.
- 선택 결과를 전역 상태에 반영한다.
### rules
- 목록 UI는 FlashList를 사용한다.
- 목록 아이템은 문서 선택의 단일 진입점 역할을 한다.
- 목록에서 선택한 문서는 소개 화면과 리더 화면에서 동일하게 참조된다.
### constraints
- 문서 수가 늘어나도 목록 렌더 성능이 급격히 저하되지 않아야 한다.
- 선택 가능한 문서가 없을 때는 빈 상태 UI를 제공해야 한다.

## doc_detail
### states
- 선택된 문서의 메타데이터
- 선택된 문서 없음 상태
- 소개 화면 표시 상태
### action
- 선택된 문서의 제목, 설명, 분류 등 소개 정보를 보여준다.
- 문서가 선택되지 않았을 때 안내 문구를 표시한다.
- 리더 화면으로 이어질 문맥 정보를 유지한다.
### rules
- 소개 화면은 선택된 문서의 요약 정보를 우선 표시한다.
- 문서가 선택되지 않은 경우에도 화면은 정상 렌더링되어야 한다.
- 소개 화면의 정보는 목록에서 선택한 문서와 일치해야 한다.
### constraints
- 소개 화면이 문서 본문 전체를 대신하지 않는다.
- 선택 상태가 없을 때 잘못된 기본 문서를 임의로 노출하지 않는다.

## doc_reader
### states
- 선택된 문서 본문
- 본문 파싱 완료 상태
- 읽기 화면 로딩 상태
- 본문 없음 상태
- 리더 옵션 적용 상태
### action
- markdown 본문을 읽기 화면에 렌더링한다.
- 선택된 문서의 본문을 로드한다.
- 옵션 상태에 따라 글자 크기, 여백, 표시 방식을 반영한다.
- 본문이 없거나 선택 문서가 없을 때 대체 상태를 표시한다.
### rules
- 리더 화면은 읽기 경험을 우선하며 불필요한 조작 요소를 최소화한다.
- 본문 렌더링은 현재 선택된 문서를 기준으로 동작한다.
- 옵션 변경은 리더 화면에 즉시 반영된다.
### constraints
- 문서 선택이 없는 상태에서 잘못된 본문을 렌더링하지 않는다.
- markdown 원문이 손상되지 않도록 표시용 변환과 원본 데이터를 분리한다.

## option_tab
### states
- 글자 크기 설정
- 줄 간격 또는 여백 설정
- 테마 또는 배경 표시 설정
- 옵션 초기값 여부
### action
- 리더 관련 옵션 값을 변경한다.
- 옵션 값을 기본값으로 되돌린다.
- 현재 옵션 상태를 전역 store에 반영한다.
### rules
- 옵션 화면은 리더 동작에 직접 연결되는 설정만 다룬다.
- 옵션 값은 zustand 전역 상태로 관리한다.
- 변경된 옵션은 리더 화면에서 즉시 확인 가능해야 한다.
### constraints
- 옵션 변경이 현재 선택 문서 상태를 초기화하면 안 된다.
- 옵션 항목은 모바일 화면에서 한 손 조작 가능한 범위를 우선한다.

## reader_store
### states
- 문서 컬렉션 상태
- 선택 문서 상태
- 리더 옵션 상태
- 앱 초기화 상태
### action
- 문서 목록과 선택 상태를 저장한다.
- 리더 옵션을 업데이트한다.
- 초기 기본값을 설정한다.
- 화면 간 공유 상태를 제공한다.
### rules
- 전역 공유 상태는 zustand store를 단일 기준으로 사용한다.
- 문서 선택 상태와 옵션 상태는 명확히 분리한다.
- 파생 가능한 값은 store 중복 저장보다 계산 우선으로 관리한다.
### constraints
- 화면별 지역 상태와 전역 상태가 충돌하지 않아야 한다.
- 동일 의미의 상태를 여러 store로 중복 보관하지 않는다.

## qa_maestro
### states
- 핵심 사용자 시나리오 정의 상태
- Maestro 테스트 작성 상태
- 테스트 실행 성공 여부
- 회귀 검증 상태
### action
- 앱 실행 후 문서 목록 진입을 검증한다.
- 문서를 선택하고 소개 화면 표시를 검증한다.
- 리더 탭 이동 후 본문 표시를 검증한다.
- 옵션 변경 후 리더 화면 반영을 검증한다.
### rules
- 핵심 플로우는 Maestro 시나리오로 자동화한다.
- 테스트는 `doc list -> doc 소개 -> doc reader -> option` 흐름을 포함해야 한다.
- 테스트 시나리오는 사용자가 실제 탭 전환과 문서 선택을 수행하는 방식으로 작성한다.
### constraints
- 단위 테스트만으로 사용자 흐름 검증을 대체하지 않는다.
- 화면 요소 식별자는 Maestro에서 안정적으로 참조 가능해야 한다.
